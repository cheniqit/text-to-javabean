通用定义

时间定义

  • 除特殊说明之外，所有时间通信使用北京时间通信。格式为 14 位 yyyyMMddhhmmss ,例如20141225140000 ，如果业务无需精确到秒，根据业务需要后面填充最小/最大时间。
  • 注: 客户端需要自行处理时区问题，与服务器的通信均认为是北京时间。

session管理

客户端自身需要维护 session 管理，即使用同一个 session 的客户端进行 http 链接。

错误信息

除下载文件外，所有返回信息均为 json，必定包含 success 属性，有错误时必定包含 errorcode属性

{
    "success":“T”,
    "errcode":'-1',
    "errmsg":'错误信息'
    }

errorcode 表示:

  1. -701 session 超时或者不存在该 session
  2. -702 该 session 因为密码被修改而失效
  3. -703 该 session 因用户被禁用而失效

下载文件错误信息

下载文件时，所有错误信息使用 http 错误状态码提示:

  • 404 表示未找到文件
  • 403 表示无权限下载

请求报文Url及参数格式

http://ip:port/ots/业务类别/业务场景（动词:查询-query或querylist；创建-create）

所有 url 全部使用小写英文字母
    所有参数全部小写英文字母

事例：

http://ip:port/ots/hotel/querylist
    http://ip:port/ots/score/modify
    http://ip:port/ots/order/create
    http://ip:port/ots/order/cancel

报文中判断 true 和 false 的判断

报文中判断是否的字段，均使用（T/F），T 代表 true，F 代表 false

评分

查询酒店评价信息

业务说明： < 根据酒店id查询酒店评价信息列表,<2分归入差评，>=2且<3分归入一般，>3分归入好评.

请求方式：

GET

接口url:

http://ip:port/ots/score/scorelist

请求参数：

字段 名称 是否必须 说明
hotelId 酒店id 是
mark 用户mark 否
token 用户token 否
gradeType 评价分类 否 好：G，中：M, 差：B， 全部：A ； < 2分归入差评，>=2且<3分归入一般，>3分归入好评.
orderBy 排序项 否
page 第几页 是 当前第几页
limit 每页多少条 是
callMethod 调用来源 否 1-crs；
2-web；
3-wechat；
4-app(ios)；
5-app(Android)
callVersion 调用版本 否
ip IP地址 否
hardWareCode 硬件编码 否
otsVersion OTS版本 否

API返回json数据示例：

{
    "success":"T",
    "errCode":,//错误码
    "errMsg":,//错误信息
    "gradeType":A, //评价分类
    "list":[{//评价信息
    "mId":,//会员id
    "telphone":,//手机号
    "grade":,//评价分数
    "comment":,//评价内容
    "createTime":"201601191010",//评价时间
    "pic":,//用户图片
    "roomtype":,//房型
    }]
    "pageSize":,//总页数

    }
